## Introduction

When working on enterprise-level GIS applications, one of the most powerful tools available is the ArcGIS ecosystem.

:contentReference[oaicite:1]{index=1} provides a complete solution for building interactive, data-driven mapping applications directly in the browser.

---

## Why ArcGIS API for JavaScript?

Unlike lightweight mapping libraries, ArcGIS API is designed for **enterprise GIS systems**.

### Key strengths:
- Rich spatial analysis tools
- High-quality map rendering
- Built-in support for layers and services
- Integration with ArcGIS Online & Enterprise

---

## Core Concepts

### 1. Map & View
Every application starts with a map and a view.

- `Map` → defines layers and basemap
- `MapView` → handles rendering in 2D

---

### 2. Layers

ArcGIS supports multiple layer types:
- FeatureLayer
- MapImageLayer
- TileLayer

Each layer represents a different data source.

---

### 3. Services Integration

One of the strongest features is integration with GIS services:

- WMS
- Feature Services
- REST APIs

This makes it ideal for national-scale systems.

---

## Example Setup

```javascript
import Map from "@arcgis/core/Map";
import MapView from "@arcgis/core/views/MapView";
import FeatureLayer from "@arcgis/core/layers/FeatureLayer";

const map = new Map({
  basemap: "topo-vector"
});

const view = new MapView({
  container: "viewDiv",
  map: map,
  center: [31.2357, 30.0444],
  zoom: 10
});

const layer = new FeatureLayer({
  url: "https://sampleserver/arcgis/rest/services"
});

map.add(layer);
```
